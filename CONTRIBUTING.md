# Contributing

Thank you for your interest in contributing to this project.

This project is primarily maintained by me, so contributions are currently limited to invited collaborators. If you have not been invited, please do not open pull requests.

## Before You Start

Please open an issue before beginning any significant work. This helps prevent duplicate effort and ensures the proposed change aligns with the project's direction.

## What You Can Contribute

Contributions are welcome in the following areas:

* Bug fixes
* Accessibility improvements
* Performance improvements

If you have an idea outside these categories, discuss it in an issue before starting.

## Development Guidelines

### Branches

Create a dedicated feature branch for your work. Do not commit directly to the main branch.

### Commit Messages

Use clear, descriptive commit messages written in English.

Examples:

* `Fix mobile navigation overflow`
* `Improve keyboard navigation for theme toggle`
* `Reduce image loading time`

### Code Style

* Match the existing coding style.
* Keep changes focused and consistent with the surrounding code.
* Avoid unrelated formatting or whitespace changes.

## Pull Requests

Please keep pull requests focused on a single logical change.

For UI changes:

* Include before/after screenshots.

When appropriate:

* Update documentation alongside your changes.

## Dependencies

This project intentionally keeps dependencies to a minimum.

Do **not** add new npm packages without prior approval.

## Project Philosophy

When contributing, keep these goals in mind:

* Keep the project lightweight.
* Prefer minimal dependencies.
* Prioritize accessibility.
* Prioritize performance.
* Avoid unnecessary complexity.

## What Not to Do

Please do not:

* Redesign the entire site.
* Replace Express with another framework.
* Introduce React, Vue, Svelte, or other frontend frameworks.
* Make unrelated formatting or cleanup changes.

If you think one of these changes would genuinely benefit the project, discuss it in an issue first.

## AI-Generated Code

AI-generated code is **not permitted** in this repository.

All submitted code should be written and understood by the contributor.

## Questions

If you're unsure whether a change is appropriate, open an issue before starting work. It's much easier to discuss an idea beforehand than after a large pull request has been opened.

Thank you for helping improve the project.
